# project-amplifier

## 核心工作流

```
项目分析 → 卖点提炼 → 公众号文章 → 短视频脚本 → 分发策略
```

## 快速开始

### 1. 分析项目（输入）

用户提供：
- 项目描述/技术栈/功能
- 目标用户群体
- 期望的转化目标

### 2. 输出内容

| 格式 | 内容 |
|------|------|
| 公众号 | 标题、钩子、正文、结尾CTA |
| 短视频 | 3秒开头、痛点场景、解决方案、行动号召 |
| 分发策略 | 抖音/小红书/B站/微信 差异化调整 |

## 触发方式

用户可能这样说：
- "帮我把这个项目写成公众号文章"
- "我开发了一个XX工具，适合发什么平台"
- "这个项目值得做自媒体推广吗"
- "生成一个吸引老板的视频脚本"
- "帮我分析这个产品有什么卖点"

## 风格标准

- **语言**：中文
- **风格**：Dan Koe式 - 直接、钩子驱动、高转化
- **长度**：公众号 1500-2500字，短视频 45-60秒
